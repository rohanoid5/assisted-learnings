# 11.2 — 1D Dynamic Programming

## What Is 1D DP?

1D DP problems have a single state variable — typically position or index. The recurrence looks at a fixed number of previous states.

```
dp[i] = f(dp[i-1])          ← 1 previous state
dp[i] = f(dp[i-1], dp[i-2]) ← 2 previous states
dp[i] = max/min over j<i    ← all previous states (O(n²))
```

## Problem 1: Climbing Stairs (LC #70)

You can take 1 or 2 steps. How many distinct ways to reach step n?

```
n = 4:
  1+1+1+1
  1+1+2
  1+2+1      ← 5 ways
  2+1+1
  2+2
```

**State:** `dp[i]` = number of ways to reach step i

**Recurrence:** `dp[i] = dp[i-1] + dp[i-2]`  
(from step i-1 take 1 step, or from step i-2 take 2 steps)

```java
int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2;
    for (int i = 3; i <= n; i++) {
        int c = a + b;
        a = b; b = c;
    }
    return b;
}
```

This is Fibonacci! `dp[i] = fib(i+1)`

## Problem 2: House Robber (LC #198)

Rob houses in a row; adjacent houses have an alarm. Maximize loot.

```
houses: [2, 7, 9, 3, 1]

Rob 0, 2, 4 → 2+9+1 = 12  ✗ (4 is only +1)
Rob 1, 3    → 7+3   = 10  ✗
Rob 0, 2    → 2+9   = 11  ✗
Rob 1, 4    → 7+1   =  8  ✗
Rob 0, 2    → ....
Rob 1, 3    → ....
Best: rob 1, 2 skipped, rob 4 → wait...

Actually: dp trace
  i=0: dp = 2
  i=1: dp = max(2, 7) = 7
  i=2: dp = max(7, 2+9) = max(7, 11) = 11
  i=3: dp = max(11, 7+3) = max(11, 10) = 11
  i=4: dp = max(11, 11+1) = 12
Answer: 12  (rob houses 0, 2, 4 → 2+9+1)
```

**State:** `dp[i]` = max loot using houses 0..i

**Recurrence:** `dp[i] = max(dp[i-1], dp[i-2] + houses[i])`  
(skip house i, or rob house i skipping i-1)

```java
int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    int prev2 = nums[0], prev1 = Math.max(nums[0], nums[1]);
    for (int i = 2; i < n; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

**House Robber II (LC #213):** Houses are circular (first and last adjacent). Solve twice: rob [0..n-2] or rob [1..n-1], take max.

```java
int rob2(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    return Math.max(
        robRange(nums, 0, n-2),
        robRange(nums, 1, n-1)
    );
}

int robRange(int[] nums, int l, int r) {
    int prev2 = 0, prev1 = 0;
    for (int i = l; i <= r; i++) {
        int curr = Math.max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## Problem 3: Coin Change (LC #322)

Given coin denominations and a target amount, find minimum coins to reach amount (infinite supply of each).

```
coins = [1, 5, 6, 9], amount = 11

Greedy fails: 9+1+1 = 3 coins
Optimal:      6+5   = 2 coins ← DP finds this

dp trace (amount = 11):
  dp[0] = 0
  dp[1] = dp[0] + coin(1) = 1
  dp[5] = dp[0] + coin(5) = 1
  dp[6] = dp[0] + coin(6) = 1  or  dp[1] + coin(5) = 2 → min = 1
  dp[10]= dp[5] + coin(5) = 2  or  dp[4]+coin(6)=... 
  dp[11]= min(dp[10]+1, dp[6]+5, dp[5]+6, dp[2]+9) = min(3,2,...) = 2
```

**State:** `dp[i]` = min coins to make amount i

**Recurrence:** `dp[i] = min over each coin c: dp[i-c] + 1`

```java
int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount + 1];
    Arrays.fill(dp, amount + 1); // "infinity"
    dp[0] = 0;

    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (coin <= i) {
                dp[i] = Math.min(dp[i], dp[i - coin] + 1);
            }
        }
    }

    return dp[amount] > amount ? -1 : dp[amount];
}
```

**Variants:**
- Coin Change II (LC #518): Count number of ways → change `min` to `+=`

```java
int change(int amount, int[] coins) {
    int[] dp = new int[amount + 1];
    dp[0] = 1;
    for (int coin : coins)  // outer loop = coins → unbounded knapsack order
        for (int i = coin; i <= amount; i++)
            dp[i] += dp[i - coin];
    return dp[amount];
}
```

## Problem 4: Longest Increasing Subsequence (LC #300)

Find the length of the longest strictly increasing subsequence.

```
nums = [10, 9, 2, 5, 3, 7, 101, 18]

Subsequences:
  [2, 5, 7, 101] → length 4
  [2, 3, 7, 101] → length 4
  [2, 5, 7, 18]  → length 4  ← answer: 4
```

**O(n²) DP:**  
`dp[i]` = length of LIS ending at index i

```java
int lengthOfLIS(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    Arrays.fill(dp, 1); // LIS ending at i is at least 1 (just nums[i])
    int max = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (nums[j] < nums[i]) {
                dp[i] = Math.max(dp[i], dp[j] + 1);
            }
        }
        max = Math.max(max, dp[i]);
    }
    return max;
}
```

**O(n log n) Patience Sorting:**

```java
int lengthOfLIS(int[] nums) {
    List<Integer> tails = new ArrayList<>();
    for (int num : nums) {
        int pos = Collections.binarySearch(tails, num);
        if (pos < 0) pos = -(pos + 1); // insertion point
        if (pos == tails.size()) tails.add(num);
        else tails.set(pos, num);
    }
    return tails.size();
}
```

> `tails` is not the actual LIS — it's a structural artifact. The *size* equals the LIS length.

## Problem 5: Decode Ways (LC #91)

A string of digits can be decoded as letters (1=A, 26=Z). Count decodings.

```
"226" → 
  2,2,6  → B,B,F
  22,6   → V,F
  2,26   → B,Z
  226    → invalid (>26)
Answer: 3
```

**State:** `dp[i]` = number of ways to decode `s[0..i-1]`

```java
int numDecodings(String s) {
    int n = s.length();
    int[] dp = new int[n+1];
    dp[0] = 1;
    dp[1] = s.charAt(0) == '0' ? 0 : 1;

    for (int i = 2; i <= n; i++) {
        int oneDigit = s.charAt(i-1) - '0';
        int twoDigit = Integer.parseInt(s.substring(i-2, i));

        if (oneDigit != 0) dp[i] += dp[i-1]; // use 1 digit
        if (twoDigit >= 10 && twoDigit <= 26) dp[i] += dp[i-2]; // use 2 digits
    }
    return dp[n];
}
```

## Pattern Summary

| Problem Type | State | Recurrence Pattern |
|-------------|-------|-------------------|
| Path counting | `dp[i]` = ways to reach i | `dp[i] = dp[i-1] + dp[i-2]` |
| Optimization (no adjacent) | `dp[i]` = best using 0..i | `dp[i] = max(dp[i-1], dp[i-2]+val[i])` |
| Set coverage (coins) | `dp[i]` = min/ways for amount i | loop over choices, `dp[i] = min(dp[i-c]+1)` |
| LIS family | `dp[i]` = best ending at i | inner loop j<i, conditional update |

## Try It Yourself

**1.** Solve Max Product Subarray (LC #152). Hint: track both max and min (negative × negative = positive).

<summary>Solution</summary>

```java
int maxProduct(int[] nums) {
    int maxProd = nums[0], minProd = nums[0], result = nums[0];
    for (int i = 1; i < nums.length; i++) {
        if (nums[i] < 0) {
            int tmp = maxProd; maxProd = minProd; minProd = tmp;
        }
        maxProd = Math.max(nums[i], maxProd * nums[i]);
        minProd = Math.min(nums[i], minProd * nums[i]);
        result = Math.max(result, maxProd);
    }
    return result;
}
```
